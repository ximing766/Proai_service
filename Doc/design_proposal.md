# 求知3 Pro AI 智能安全座椅 - 交互流程与技术架构方案

## 1. 总体架构设计

本方案基于 Linux 平台的 AI 模组，采用 **主服务 (Master Service)** 与 **从服务 (Slave Service)** 分离的架构设计，以实现业务逻辑与硬件通信的解耦，提高系统的稳定性和可维护性。

### 1.1 硬件架构
- **AI 模组**: 核心计算单元，运行 Linux 系统。集成 Cat1 (4G), 蓝牙, GPS, 音频 (Mic/SPK)。
- **安全座椅控制主板 (MCU)**: 负责座椅的具体执行（加热、通风、姿态）和传感器数据采集。
- **连接方式**: AI 模组与 MCU 通过 **UART 串口** 连接，运行涂鸦串口协议。

## 2. 详细实现方案

### 2.1 模块划分与职责

#### 2.1.1 从服务 (Slave Service / `uart_service`)
**定位**: 硬件抽象层，作为 MCU 的代理，屏蔽串口通信细节。
- **核心功能**:
  - **串口通信**: 负责 UART 的读写，维护波特率和缓冲区。
  - **协议解析**: 实现涂鸦串口协议 (Sections 7, 8, 9)，处理帧头、校验和、转义。
  - **心跳维护**: 独立与 MCU 进行心跳保活 (0x00)，断连时尝试重连或报警。
  - **DP 数据透传**: 将解析后的 DP (Data Point) 数据通过 IPC 发送给主服务；接收主服务的 DP 指令，将其封装为涂鸦串口协议帧发送给 MCU。
  - **OTA 桥接**: 接收主服务的 MCU 升级包，按协议分包发送给 MCU。
- **技术栈**: C/C++ (推荐，低延迟) 或 Python (开发快)。建议使用 `pyserial` 或 C `termios`。

#### 2.1.2 主服务 (Master Service / `main_service`)
**定位**: 系统的“大脑”，处理所有业务逻辑、云端交互和人机交互。
- **核心功能**:
  - **IoT 管理**: 集成涂鸦 SDK 或 MQTT 客户端，处理配网、设备状态上报、App 指令接收。
  - **AI 交互**: 
    - **ASR (语音转文字)**: 采集 Mic 数据，进行本地关键词检测 (Wake Word) 和云端识别。
    - **NLP/LLM**: 将识别文本发送至 AI 大模型，获取回复和指令 (如“打开座椅通风”)。
    - **TTS (文字转语音)**: 将 AI 回复播放出来。
  - **业务逻辑**: 状态机管理（待机、唤醒、对话中、升级中）。
  - **IPC 客户端**: 通过 IPC 订阅 MCU 状态，发送控制指令。
- **技术栈**: Python (生态丰富，适合 AI/HTTP 逻辑)。

### 2.2 交互流程详解

#### 2.2.1 初始化流程
1. **Slave 启动**: 打开串口，向 MCU 发送心跳 (0x00)。
2. **握手成功**: MCU 回复心跳，Slave 查询产品信息 (0x01) 和工作模式 (0x02)。
3. **Master 启动**: 初始化音频、网络。
4. **IPC 连接**: Master 连接 Slave 的 IPC 接口。
5. **状态同步**: Master 请求 Slave 获取 MCU 当前所有 DP 状态 (0x08 -> 0x07)，并上报云端。

#### 2.2.2 语音控制流程 (AI 闭环)
1. **用户**: "你好小知，打开座椅通风。"
2. **AudioMgr**: 唤醒词检测成功，启动云端 ASR。
3. **AI Agent**: 识别意图 -> `Intention: OPEN_VENTILATION`。
4. **BizLogic**: 
   - 构造指令: `DPID_VENTILATION = True`。
   - 发送给 Slave (IPC)。
   - 生成 TTS: "好的，已为您打开通风。" -> AudioMgr 播放。
5. **Slave Service**: 收到 IPC 指令，封装成涂鸦协议帧 (0x06 命令下发) -> UART -> MCU。
6. **MCU**: 执行动作，开启风扇。回复 ACK 或 状态上报 (0x07)。
7. **Slave Service**: 收到 0x07，解析后通过 IPC 通知 Master。
8. **Master**: 更新内存状态，上报云端。

#### 2.2.3 OTA 升级流程
1. **CloudMgr**: 收到云端 MCU 升级通知 (URL, MD5, Size)。
2. **Master**: 下载固件包。
3. **Master -> Slave**: 发送“开始升级”指令 (IPC)。
4. **Slave -> MCU**: 发送 0x0A (升级启动)。
5. **MCU**: 回复同意。
6. **Slave**: 将固件分包，循环发送 0x0B (包传输)。
7. **MCU**: 收包并校验，写入 Flash。
8. **Slave**: 监控进度，上报 Master。
